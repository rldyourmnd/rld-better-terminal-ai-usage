# v1.0.0 Answer Lock (RLDYourTerm)

## Status
- v1.0.0 requirements lock: finalized.
- Дата фиксации: 2026-02-24.
- Все вопросы из `planning/discovery/v1.0.0-requirements-questionnaire.md` закрыты для v1.0 scope.

## Вопросы и решения (1..22)

1) Основной пользователь
- Ответ: продуктовый владелец и AI/CLI power-user с фокусом на визуал и стабильность длительных сессий.

2) Тип задач
- Ответ: mix use case, с упором на manual work + AI пайплайны, с длительными фоновыми и интерактивными сессиями.

3) SLA по падениям
- Ответ: минимальные падения. Норматив: любая ошибка в GPU не должна завершать сессию; допускается controlled fallback.

4) Политика автоматического fallback
- Ответ: автоматический переход с уведомлением. Любой `gpu`-failure должен вызывать событие и user notification.

5) RAM-пороги
- Ответ: не задаём жёсткий порог в проектном плане; приоритет — минимизация по мере оптимизации. Будет применяться профилирование и лимиты буферов по умолчанию.

6) FPS целевой
- Ответ: нет фиксированного числа как обязательного порога. Цель — отсутствует конфликт при смене мониторов (144Hz/60Hz), плавность и минимальная задержка.

7) Shell сценарии
- Ответ: дефолт `fish + Starship`, fallback `zsh`, Bash не включён как обязательный v1.0 target на Linux/macOS.

8) Legacy совместимость
- Ответ: приоритет стабильности и производительности базовых сценарием; поддержка legacy ANSI расширений реализуется с деградацией без падения сессии.

9) Критичные события для AI-инструментов
- Ответ: prompt latency, copy/paste latency, сохранность scrollback, корректный cursor/color state, отсутствие color bleed.

10) Scrollback
- Ответ: обязательный минимум 50_000 строк с hard cap (включает политики очистки для ограничения RAM).

11) Мульти-окна в v1
- Ответ: минималистично — один терминал/одно окно.

12) UI-стиль
- Ответ: базовый профиль `cuberpunk`, гибкие профили проектируются, но в v1 добавляем только базовый.

13) Конфигурация GPU по платформам
- Ответ: используем backends, подходящие для платформы через `wgpu`: Linux/Intel/AMD/NVIDIA -> Vulkan/OpenGL по возможности, macOS -> Metal, fallback внутри `wgpu` без ручного переключения.

14) Текстовые эффекты
- Ответ: чистый рендер в v1.0 без дорогостоящих визуальных эффектов (blur/shadow/gradients в roadmap).

15) Windows

- Ответ: архитектурная заготовка включена в foundation, функциональная полнота не входит в v1.0.

16) Наблюдаемость
- Ответ: structured logs, frame timing, memory trend, event-id, GPU/PTY diagnostics, с отдельным debug mode.

17) Метрики для релиза
- Ответ: в release baseline включаются минимум:
  - startup readiness (fish+starship paint + prompt readiness),
  - prompt-to-paint latency (p95),
  - copy/paste latency (p95),
  - frame time и CPU/GPU load,
  - crash-free runtime доля,
  - bounded memory growth rate.

18) Распространение
- Ответ: primary release через GitHub releases + install script; план по apt/homebrew в v1.0 roadmap как внешний канал установки.

19) Профили производительности
- Ответ: поддерживаются в рамках ручного тест-плана: cpu-only, gpu-only, auto.

20) Блокирующие баги
- Ответ: блокеры: падение старта shell, постоянные падения, деградация prompt/цветов/скролла, memory leak, частые GPU crashes.

21) Расширения в roadmap в дизайне
- Ответ: фиксация планов через backlog: session recorder, split-screen, advanced diagnostics и т.д. в пост-v1 блоке, архитектурно подготовлены hooks.

22) Формат документации для CI
- Ответ: в v1.0 CI не используется, поэтому все спецификации и acceptance criteria пишутся в markdown с четкими вручную-валидируемыми чек-листами.

## Rule
