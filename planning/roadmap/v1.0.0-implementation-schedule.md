# v1.0.0 Implementation Schedule (step-by-step)

## Спринт 0 — Documentation lock (now)
- Lock all discovery answers.
- Verify all required ADRs are `Accepted`.
- Freeze architecture, stack and risk scopes for coding.
- Prepare folder contract per OS (`linux`, `macos`, `windows-skeleton`).

## Спринт 1 — Foundation adapters
- Implement PTY adapter interface backed by `portable-pty`.
- Implement window event adapter backed by `winit`.
- Implement diagnostics facade (`tracing`, `event-id`, structured output).
- Deliverables:
  - foundation module contracts and adapter tests.
  - initial `render_mode` service boundaries.

## Спринт 2 — Core runtime
- Implement minimal terminal state model in `core`.
- Implement parser boundary and cursor/grid transitions for baseline ANSI set.
- Implement services lifecycle for spawn/read/write/resize/teardown.
- Deliverables:
  - session lifecycle orchestration,
  - bounded transition + fallback path,
  - deterministic shutdown.

## Спринт 3 — Features baseline
- Implement command palette and in-terminal settings commands.
- Implement runtime profiles/settings persistence (typed local profile).
- Implement fish/starship auto-init with zsh fallback.
- Deliverables:
  - shell service states (default/fallback/fail-safe),
  - settings history + rollback logic,
  - debug mode for diagnostics.

## Спринт 4 — Render dual-stack
- Implement CPU renderer baseline.
- Implement GPU renderer via `wgpu` backend abstraction.
- Implement `cpu/gpu/auto` service selection and failure counters.
- Deliverables:
  - render error policy with thresholds,
  - user-visible notifications,
  - benchmarkable mode switching.

## Спринт 5 — Staging/stability hardening
- Long session and burst stress pass (manual + scripted smoke).
- resize storm on multi-monitor transitions.
- Memory hard caps and scrollback retention checks.
- Deliverables:
  - updated risk matrix with closure,
  - completed `planning/operations/v1.0.0-manual-test-plan.md`.

## Спринт 6 — Release pack and freeze
- Manual release checklist and sign-off.
- final docs lock update (`1.0.0` markers and version references).
- package/release preparation for GitHub artifacts + installer script plan.
- Deliverables:
  - `metrics/version/1.0.0/definition_of_done.md`,
  - final `planning/operations/v1.0.0-release-pack.md` execution, no CI.

## Completion Criteria
- все этапы закрываются только после выполнения ручного acceptance matrix.
- все изменения в runtime-сервисах должны ссылаться на ADR и stack contracts.
